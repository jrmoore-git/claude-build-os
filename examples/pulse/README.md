# Pulse -- Worked Example

This is a worked example showing what Build OS artifacts look like for a real project. Use it as a reference when filling in your own.

**The project:** Pulse is a weekly team health check tool delivered via Slack. Engineering team leads send 3-question pulse surveys to their reports every Monday, collect 1-5 scale responses, and view trends on a dashboard.

**What's here:**

| File | What it shows |
|---|---|
| `docs/project-prd.md` | A complete PRD with all 9 sections filled in |
| `docs/current-state.md` | Mid-project state after the first sprint |
| `tasks/decisions.md` | 3 architectural decisions with rationale |
| `tasks/lessons.md` | 2 lessons learned during implementation |

**How these artifacts were created:**

1. `/think discover` produced the design doc (not included here -- the PRD supersedes it for this example)
2. PRD was generated from the design doc conversation in Phase 6.5
3. Decisions were recorded as they came up during `/challenge` and `/plan`
4. Lessons were recorded when things broke during implementation
5. `current-state.md` was written by `/wrap` after the first sprint
