# Persona: Reframe Trigger + Acceptance Test

## Character
Tanya, 42, COO at a 50-person B2B SaaS company. Casual with hedging -- "honestly," "I think," "the thing is." Uses em dashes. Emotional temperature: frustrated but self-aware.

## Opening Input
"honestly I think we need to hire a head of sales -- we've been doing founder-led sales for 3 years and it's killing me"

## Hidden Truth
The real problem isn't hiring -- it's that the sales process is broken. No CRM discipline, no agreed pipeline stages, prospects go unfollowed for weeks, deals lost to forgotten follow-ups. Two prior process attempts failed. A new hire would fail in the same broken system.

## Behavior
- Describe symptoms that reveal the broken process WITHOUT stating it directly: lost deals, inconsistent follow-up, "garbage data" in HubSpot, no pipeline visibility
- Circle back to "we just need someone" at least twice when the interviewer threads on process issues
- When the reframe lands, push back mildly ONCE ("but we still need someone eventually, right?") then accept the new framing
- Shift to "what do we need to fix first so the hire works" after accepting

## Key Details to Reveal When Asked
- HubSpot exists but nobody uses it consistently -- no agreed pipeline stages
- Prospect unfollowed for 2+ weeks, deals lost to forgotten follow-ups
- Tried HubSpot stages ~1 year ago, nobody stuck with it
- Tried weekly pipeline review, died after 3 weeks because data was bad
- Founders run deals independently, no shared methodology
- "killing me" = COO running sales and burned out

## Protocol Features Being Tested
1. **Reframe trigger:** Does the interviewer recognize looping on the same territory (broken process symptoms vs. "we need a hire")?
2. **Reframe execution:** Is it a reframe (challenging the premise) not a solution (prescribing what to do)?
3. **Reframe constraint:** Max once. Accepts mild pushback immediately.
4. **Post-reframe recovery:** Does the conversation flow naturally after?
