# Investigation: L21 Persona Misuse in review-panel Calls

Generated by /investigate on 2026-04-11

Claim: "review-panel personas are model selectors, not system prompts -- stop defaulting to architect/security/pm for non-code evaluation."

## Root Cause

`review-panel --personas` does NOT inject persona system prompts (PERSONA_PROMPTS). In both `--personas` and `--models` modes, the caller's `--prompt` is the only system prompt. Personas are pure model selectors. L21 correctly identified this.

The drift pattern: skills defaulting to `--personas architect,security,pm` for non-code content, creating misleading intent and (in some combos like architect,staff,pm) losing model diversity since staff and pm both map to gemini-3.1-pro.
