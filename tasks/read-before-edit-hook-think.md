# Brief: Read-Before-Edit Enforcement Hook (D9 Phase 2)

Generated by /think refine on 2026-04-15

## Problem Statement
Advisory rule "Inspect before acting" in CLAUDE.md fails under context pressure (L25, L21). Claude edits files it hasn't read, causing confident-but-wrong changes. The deferral blocker ("PreToolUse hook infrastructure doesn't exist yet") is resolved — 17 hooks exist, including hook-pre-edit-gate.sh on the exact same PreToolUse Write|Edit path.

## Proposed Approach
Simple two-part hook:
1. **PostToolUse Read tracker:** Records which files have been Read in the current session to a temp tracking file (`/tmp/buildos-read-tracker-{SESSION_ID}.json`)
2. **PreToolUse Edit/Write gate:** On Edit or Write, checks the tracking file. If the target file hasn't been Read this session, emit a warning (not a hard block).

Scope: same protected paths as `hook-pre-edit-gate.sh` (from `config/protected-paths.json`). Exempt paths (tasks/, docs/, tests/, config/, stores/) pass through.

## Key Assumptions
- Warning-only is sufficient to start (not blocking). Promote to blocking after validation if warnings are frequently ignored.
- New-file creation is exempt — files that don't exist yet can't be Read first. Detect via `[ ! -f "$FILE_PATH" ]`.
- Grep/Glob matches count as "awareness" of a file — not just Read. Track Grep results that include the target file path.
- Session tracking via temp file keyed on `$CLAUDE_SESSION_ID` (or parent PID fallback, same as decompose gate).

## Risks
- False positives on new files — mitigated by new-file exemption
- Stale tracking file from prior session — mitigated by session ID scoping
- Performance: extra file read on every Edit/Write — negligible (JSON parse of a small file)

## Simplest Testable Version
Single Python hook (~50 lines) that:
1. On PostToolUse Read: append file_path to session tracking file
2. On PreToolUse Write|Edit: check if file_path is in tracking file AND file is on a protected path
3. If not read and protected: warn (exit 0 + advisory message), don't block
4. If not read and not protected: pass silently
5. New files (don't exist on disk): pass silently

## Future consideration
If this pattern recurs (read-before-review, read-before-delete), extract into a generic tool-observation tracker framework. Not building that now — ship the simple hook, revisit with data.
