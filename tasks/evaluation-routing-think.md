# Brief: Content-aware evaluation routing

Generated by /think refine on 2026-04-11

## Problem Statement
The system repeatedly uses code-review personas (architect/security/pm) for non-code evaluation because there's no structural enforcement matching evaluation approach to content type. Advisory lessons don't survive execution pressure — L21 was violated within one turn of being written.

## Proposed Approach
Two changes:

1. **Add `--models` to `review-panel`** — allows callers to specify models directly without going through persona names. When `--models` is used instead of `--personas`, no persona system prompt is injected (the caller's `--prompt`/`--prompt-file` is the only system prompt). This removes the misleading indirection where "architect" means "claude-sonnet" but sounds like "architecture reviewer."

2. **Update `/review` skill content-type routing** — D15 already detects code vs. document. Extend it: code routes to `review-panel --personas` (persona framing appropriate). Documents route to `review-panel --models` with a caller-provided evaluation prompt (no persona framing). The routing decision happens in the skill, not in the caller's head.

## Key Assumptions
- The persona system prompts in `review-panel` are only useful for code review, not document evaluation (verified: PERSONA_PROMPTS are adversarial code-review prompts)
- `--prompt`/`--prompt-file` already serves as the system prompt in review-panel (verified: line 2889 of debate.py)
- The `/review` skill is the primary entry point for evaluation (if people call debate.py directly, they bypass the routing — but the skill is the recommended path)

## Risks
- People calling `debate.py review-panel` directly still need to think about persona fit. The skill routing helps but doesn't prevent direct misuse.
- Adding `--models` makes `--personas` and `--models` mutually exclusive — need to enforce that clearly.

## Simplest Testable Version
1. Add `--models` arg to review-panel subparser (mutually exclusive with `--personas`)
2. When `--models` is used, skip persona lookup — just map each model string directly
3. Update `/review` SKILL.md: document content -> `--models claude-opus-4-6,gemini-3.1-pro,gpt-5.4 --prompt-file <eval-prompt>`
4. Test: run a document evaluation through `/review` and confirm no persona framing leaks
